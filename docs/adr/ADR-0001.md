# ADR-0001: ADR 模板（起始条目）

- Status: Accepted
- Date: 2026-03-13
- Owner: paperrss maintainers

## 背景

本项目需要长期演进；若没有结构化决策记录，后续改动容易重复讨论且难以回溯。

## 决策

采用标准 ADR 模板并要求所有关键设计变更沉淀到 `docs/adr/`。

## 影响

- 正向：决策可追溯，AI 代理可直接引用历史约束。
- 成本：每次关键变更需额外维护 ADR。

## 备选方案

1. 只在 PR 描述记录决策：检索成本高，长期不可维护。
2. 只在 README 记录：会造成 README 持续膨胀。

## 回滚策略

若 ADR 流程过重，可先将“必须写 ADR”的范围收敛到接口变更与状态 schema 变更。
