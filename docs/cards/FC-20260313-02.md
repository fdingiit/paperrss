# FC-20260313-02: 新增 `-status` 命令（轻量状态摘要）

- Card ID: `FC-20260313-02`
- Status: Accepted
- Owner: fdingiit
- Created At: 2026-03-13
- Updated At: 2026-03-13

## 背景

现有 `-ping` 返回信息较全，适合排障，但日常快速查看状态时信息偏多。需要一个更轻量的命令用于“秒级判断服务是否健康、日报/周报是否按计划运行”。

> 说明：本卡为 **vibe coding 流程演示样例（demo pack）**，用于演示工件与门禁链路；本次不要求落代码。

## 决策

### 目标与非目标

- 目标：
1. 提供 `-status` 命令，输出 1 条轻量状态卡片。
2. 卡片至少包含：整体状态、今日日报状态、本周周报状态、下次触发时间。
3. 命令支持在线程内回复（遵循 `cmd_reply_in_thread`）。
- 非目标：
1. 不替代 `-ping` 的完整诊断信息。
2. 不改变现有 `-ping/-brief/-force/-help` 行为。

### FR（功能需求）

| 编号 | 需求描述 | 实现点映射 |
| --- | --- | --- |
| FR-001 | 支持 `@bot -status` 命令识别 | `slack_cmd_toolkit.parse_command` + command router |
| FR-002 | 输出轻量状态卡片（摘要字段） | `slack_cmd_toolkit.build_command_response` |
| FR-003 | 当 `/healthz` 不可达时返回降级文本 | `build_command_response` 异常分支 |
| FR-004 | 命令回复遵循线程策略 | `app_daemon.socket_mode_loop` 现有 `cmd_reply_in_thread` |

### 风险与回滚

- 风险：命令文案重复或与 `-ping` 语义重叠，导致使用混淆。
- 回滚策略：若反馈不佳，移除 `-status` 路由并保留 `-ping`。
