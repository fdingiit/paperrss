## 背景

当前项目已有系统文档（01~06），但缺少“从需求到验收到持久化”的统一执行流程，导致每次开发在工件与门禁上标准不一致。

## 决策

### 目标与非目标

- 目标：
1. 在 `docs/` 根目录建立固定流程工件路径。
2. 提供卡片、E2E、验收三类模板。
3. 固化 G1-G6 门禁并写入流程规范。
- 非目标：
1. 不引入 GitHub CI 强制门禁。
2. 不改动运行时代码逻辑。

### FR（功能需求）

| 编号 | 需求描述 | 实现点映射 |
| --- | --- | --- |
| FR-001 | 建立 `docs/cards`、`docs/e2e`、`docs/acceptance`、`docs/templates` 目录 | 文档目录结构创建 |
| FR-002 | 提供流程总文档，定义 G1-G6 门禁 | `docs/process-vibe-coding.md` |
| FR-003 | 提供 Card/E2E/Acceptance 模板 | `docs/templates/*.md` |
| FR-004 | 建立功能级 changelog 并能追溯 Card ID | `docs/changelog.md` |

### NFR（非功能需求）

| 编号 | 需求描述 | 验证方式 |
| --- | --- | --- |
| NFR-001 | 文档可追溯性 | `Card ID` 可在四类工件中检索 |
| NFR-002 | 可执行性 | 新人可按模板独立走完流程 |
